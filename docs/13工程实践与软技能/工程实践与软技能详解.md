# 工程实践与软技能面试指南

评分：5=必须掌握，4=很重要，3=熟练更佳，2=了解即可，1=加分项

## 1. 需求澄清与价值对齐 [5]
- 方法：用户旅程 → 用例 → 非功能 (性能/安全/可用性) 明确。
- 产物：问题陈述、验收标准 (Given-When-Then)、范围边界与不做列表。
- 技巧：避免隐式假设；用示例与反例确认。
- 风险：需求飘移、过度设计；建立变更评审流程。

## 2. 架构思考与权衡 (Trade-off) [5]
- 维度：一致性/性能/成本/复杂度/可维护性。
- 评估框架：Quality Attribute Scenarios；约束优先级排序。
- 决策：列备选、标准化比较矩阵、记录否决理由。
- 反模式：银弹/过度追新技术/未验证假设。

## 3. 设计原则与演进 (SOLID/DRY/KISS) [5]
- 单一职责与聚合变化点；开闭原则控制扩展面。
- 依赖倒置：端口适配器 (Hexagonal/Clean Architecture)。
- KISS/最小可行；避免过早抽象与层级过深。
- DRY 与 “抽象腐化” 平衡：重复 vs 难懂。

## 4. 模型与领域驱动 (DDD) [4]
- 通用语言 (Ubiquitous Language)、限界上下文。
- 上下文映射：合作、伙伴、防腐层 (ACL)。
- 聚合边界：事务一致性与修改范围。
- 事件驱动：领域事件与集成事件区分。

## 5. 架构文档与 ADR (Architecture Decision Record) [4]
- 模板：Context → Decision → Status → Consequences。
- 版本化与检索；避免口头决策遗失。
- 最佳实践：只记录影响全局或不可逆决策。
- 演进：状态从 Proposed → Accepted → Deprecated。

## 6. 技术债务识别与偿还策略 [4]
- 分类：结构性/性能/安全/流程债务。
- 指标：修改频率 + 缺陷密度 + 复杂度热区。
- 策略：重构预算 (每迭代固定比例)，分阶段偿还。
- 风险：无度量 → 任意争议；引入可视化看板。

## 7. 重构与演进式设计 [4]
- 触发：重复出现、复杂度跃迁、边界不清。
- 步骤：安全测试基线 → 小步提交 → 行为保护。
- 技巧：提炼函数、抽象类/接口、封装条件、多态替换分支。
- 指标：圈复杂度下降、认知负荷降低。

## 8. 代码评审 (Code Review) [4]
- 关注点：正确性 > 边界条件 > 可读性 > 安全/性能热点。
- Anti-pattern：唯风格、过度微优化、迟到批量评论。
- 工具：Diff + 评论分组；自动检查 (lint/安全) 前置。
- 心理：建设性、聚焦行为而非个人。

## 9. 测量与工程指标体系 [4]
- 研发效能：循环时间 (Lead Time)、部署频率、失败恢复时间、变更失败率 (DORA)。
- 质量：缺陷密度、回归频率、测试差异覆盖、重构收益。
- 业务：功能到达率、错误率、延迟分位。
- 数据驱动：指标 → 假设 → 实验 → 学习闭环。

## 10. 风险管理与缓解 [4]
- 分类：需求/技术/依赖/安全/人员。
- 量化：概率 × 影响矩阵；优先处理“高概率高影响”。
- 预案：降级方案、回滚路径、应急联系人。
- 演练：定期验证关键组件失效场景。

## 11. 发布与变更管理 [3]
- 策略：金丝雀/灰度/蓝绿；特性开关配合数据指标。
- 变更窗口与冻结期；紧急变更快速审批。
- 回滚：不可变版本 + 数据兼容校验。
- 变更记录：时间戳/操作者/内容/票据。

## 12. 沟通与跨团队协作 [4]
- 清晰结构：结论先行 (BLUF)；数据支持。
- 工具：会议纪要/行动项负责人与截止日期。
- 期望管理：可交付范围 + 里程碑。
- 冲突处理：聚焦问题与事实，避免立场固化。

## 13. 影响力与技术倡导 [3]
- 推动标准：示例代码 + 模板仓库。
- 证明价值：度量提升与故障减少。
- 扩散渠道：技术分享/读书会/内部文档。

## 14. 指导与人才培养 (Mentoring) [3]
- Pair/Code Walkthrough；反馈具体且及时。
- 成长路径：技能矩阵 + OKR。
- 防止知识孤岛：轮岗与交叉评审。

## 15. 决策与取舍透明度 [3]
- 公示决策与备选方案。
- 失败复盘保留历史，降低重复试错。
- 价值对齐：业务 KPI → 技术目标映射。

## 16. 会议效率与异步协作 [3]
- 议程提前；必要人员原则。
- 决策记录替代重复会议。
- 异步文档 → 评论收敛 → 最终会确认。

## 17. 优先级管理与规划 [3]
- 四象限：紧急/重要；减少过度紧急任务。
- Rolling Wave 规划：近端细化，远端粗粒度。
- 缓冲：风险任务预留时间。

## 18. 数据隐私与伦理意识 [2]
- 最小收集原则；隐私影响评估。
- 避免滥用跟踪与不透明算法。
- 合规协作：法务/安全团队。

## 19. 自我驱动与持续学习 [3]
- 学习循环：实践→反馈→总结→改进。
- 技术雷达：跟踪成熟度与风险。
- 过载管理：限制并行学习主题。

## 20. 面试常见软技能问答速览
1. 如何处理需求频繁变动？冻结迭代边界，变更进入下周期并记录影响。  
2. 技术债务什么时候先还？影响交付速度或质量风险指数升高时优先。  
3. 代码评审的核心关注？正确性与边界，再到可读性与风险。  
4. 为什么要写 ADR？确保决策可追溯与未来演进依据。  
5. 如何说明一个技术取舍？列备选 + 评价维度 + 数据/风险 + 推荐方案。
